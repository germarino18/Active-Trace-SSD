## Why

The backend domain features for the PROFESOR role (grade import via C-10, overdue analysis via C-11, communications via C-12) are complete. C-21 established the frontend shell (auth, layout, routing, HTTP client). C-22 delivers the PROFESOR's primary workspace: import grades from the LMS, detect overdue students, export uncorrected deliveries, communicate with at-risk students, and monitor progress. Without this, the platform is unusable for its primary user role.

## What Changes

- **New `features/academico/` module** with 5 sub-features following the feature-based pattern from C-21
- **Import flow page**: file upload, activity detection preview, activity selection, threshold configuration (consumes `C-10` endpoints)
- **Overdue students view**: filtered table of at-risk students with ranking, final grades, quick reports (consumes `C-11` endpoints)
- **Uncorrected deliveries**: table of submitted-but-ungraded work + export button
- **Communication panel**: student selection, message preview (generated by backend), send with queued status tracking in real-time (consumes `C-12` endpoints)
- **Follow-up monitor**: filterable view of student activity status for TUTOR/PROFESOR roles
- **New routes** in `App.tsx` under the existing `AuthGuard / AppLayout` shell
- **Sidebar menu items** for PROFESOR added to the permission-based navigation
- **Test suite**: import flow (mocked), overdue table, communication preview, status tracking

### What is OUT of scope (deferred to C-23, C-24)
- COORDINADOR/ADMIN features (equipos, avisos, finanzas)
- E2E tests (frontend tests use mocked API)
- Any backend changes

## Capabilities

### New Capabilities

- `importar-calificaciones`: Grade import page — file upload for LMS-exported spreadsheet, activity detection preview, activity selection toggle, threshold percentage configuration
- `vista-atrasados`: Overdue students view — filtered table of students below threshold, ranking by approved activities, final grades aggregation, quick report metrics
- `entregas-sin-corregir`: Uncorrected delivery detection — table of submitted-but-unscored activities identified by crossing completion report vs grades, export to file
- `comunicacion-atrasados`: Communication to overdue students — student multi-select, message preview with personalized content, send with queue, real-time status tracking (Pending → Sending → OK/Fail/Cancelled)
- `monitor-seguimiento`: Follow-up monitor — filterable student activity status view for TUTOR and PROFESOR, with filters for student, commission, activity, minimum completion

### Modified Capabilities

*(None — first frontend domain module, no existing frontend specs to modify)*

## Impact

- New `frontend/src/features/academico/` directory with `{components,hooks,services,types,pages}` sub-folders
- New routes added to `App.tsx` within the `AuthGuard / AppLayout` hierarchy (`/materias/:id`, `/materias/:id/importar`, `/materias/:id/atrasados`, `/materias/:id/comunicar`, `/materias/:id/monitor`)
- Sidebar menu configuration updated with PROFESOR menu items (requiring `calificaciones:*`, `atrasados:*`, `comunicacion:*` permissions)
- Tests: Vitest + React Testing Library with mocked Axios for import flow, overdue table, communication preview, status tracking
- No backend changes required — consumes existing `C-10`, `C-11`, `C-12` endpoints
